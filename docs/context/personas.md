# Government User Personas

## Key Personas

| Role | Tech Fluency | Primary Goals | Main Frustrations |
|------|-------------|---------------|-------------------|
| **Budget Director** | Mid-Tech | Strategy, cross-department coordination | Excel bloat, manual tasks |
| **Maintenance Worker** | Low-Tech | Get tasks, close them on mobile | Paper work orders, poor UIs |
| **Permit Reviewer** | Mid-Tech | Queue management, compliance | Cluttered UIs, slow search |
| **Procurement Agent** | High-Tech | Speed + compliance | Manual RFPs, low supplier turnout |
| **City Manager** | Mid-Tech | Oversight, public transparency | Data silos, slow insights |
| **IT Director** | High-Tech | Integration, security, uptime | Shadow IT, system sprawl |

## Design Implications
- **Low-Tech users**: Require simplified interfaces, clear visual hierarchy, minimal cognitive load
- **Mid-Tech users**: Need efficient workflows with progressive disclosure of advanced features
- **High-Tech users**: Benefit from keyboard shortcuts, bulk operations, and integration capabilities
- **Mobile-first for field workers**: Maintenance workers and inspectors need mobile-optimized interfaces
